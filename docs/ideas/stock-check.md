# Smart Signage — เครื่องมือเคลียร์/ตรวจนับสต็อกรายตัว

## Problem Statement

**How Might We** ให้ทีม 2-3 คนเคลียร์สต็อกเครื่อง Smart Signage ที่รับช่วงต่อมาแบบไม่เคลียร์ — ให้รู้ชัดว่ามีกี่ตัว แต่ละตัวสถานะอะไร (อยู่ในสต็อก/เช่าซื้อ-ซื้อขาด/โครงการทดสอบ/ส่งซ่อม-เสีย-สูญหาย) S/N ตรงกล่องไหม อุปกรณ์ครบไหม — **ได้ไว และหลายคนทำพร้อมกันโดยไม่แก้ทับกัน**

## Recommended Direction

**ติดตามทรัพย์สิน "รายตัว" (serialized asset) + workflow การตรวจยืนยัน (audit).** ของในสต็อกคือเครื่องที่แต่ละตัวมีตัวตนเฉพาะ (S/N, MAC, ขนาดจอ 18.5/24/32/43") จึงเก็บ "1 แถว = 1 เครื่องจริง" ไม่ใช่ของกองรวมนับจำนวน

เริ่มจาก **import Excel กลางเดิม** (S/N, MAC, ขนาดจอ) เป็นจุดตั้งต้น แล้วทีมเดินตรวจยืนยันรายตัว: เปิดเครื่อง เช็คว่า S/N จริงตรงกับที่พิมพ์บนกล่องไหม อุปกรณ์ (สาย AC / ขาตั้ง-เสา / เสา Wi-Fi) ครบไหม แล้วตั้งสถานะการตรวจ (ยังไม่ตรวจ / ตรวจแล้ว / **พบปัญหา**) ระบบรวม "ตัวที่มีปัญหา" ไว้หน้าเดียวเพื่อตามเคลียร์

แนวคิดเดิม "กันการแก้ชนกัน" ยังอยู่ แต่ย้ายมาระดับรายตัว: ทุกการแก้เขียน **ChangeLog** (ใคร/แก้อะไร/เมื่อไร) + **optimistic lock** (เตือน 409 เมื่อมีคนแก้ก่อน) — **ไม่ทำ realtime websocket** เพราะทีมเล็กไม่คุ้มความซับซ้อน UI **mobile-first** กรอกผลตรวจได้เร็วด้วยมือเดียว

## Key Assumptions to Validate

- [ ] Excel เดิม map เข้าฟิลด์ S/N/MAC/ขนาดจอ ได้ตรง — _ทดสอบ: ขอหัวคอลัมน์จริงมา map ก่อนเขียน import_
- [ ] checklist อุปกรณ์ 3 อย่าง (สาย AC / ขาตั้ง-เสา / เสา Wi-Fi) ครอบคลุมพอ — _ทดสอบ: ลองตรวจจริง 2-3 กล่อง_
- [ ] "เก็บแค่สถานะพอ" (ไม่ผูกลูกค้า/โครงการ) เพียงพอรอบนี้ — ยืนยันแล้วจากผู้ใช้
- [ ] ทีม 2-3 คนตรวจพร้อมกัน optimistic lock + audit log พอ ไม่ต้อง realtime

## MVP Scope

**ทำ (In):** ล็อกอิน(admin/staff) · import Excel/CSV · รายการ+ค้นหา/กรอง(S/N,MAC,ขนาดจอ,สถานะ,สถานะตรวจ) · ฟอร์มตรวจยืนยันรายตัว(S/N เครื่อง vs กล่อง + อุปกรณ์ครบ) · auto-flag discrepancy · หน้ารวม "เครื่องมีปัญหา" · ChangeLog + optimistic lock · mobile-first

**ไม่ทำ (Out):** ดูหัวข้อถัดไป

## Not Doing (and Why)

- **ผูกลูกค้า/โครงการรายตัว** — ผู้ใช้ยืนยัน "เก็บแค่สถานะพอ" รอบนี้
- **Realtime websocket** — ทีมเล็ก optimistic lock + audit log พอ
- **บาร์โค้ด/QR สแกน** — ยังไม่มีฮาร์ดแวร์ เพิ่มทีหลัง
- **หลายสาขา/หลายคลัง · รายงาน/กราฟ · แจ้งเตือนสต็อกต่ำ · POS** — นอกโจทย์รอบเคลียร์
